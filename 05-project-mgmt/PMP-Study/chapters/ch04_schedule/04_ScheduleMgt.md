## Project Schedule Mgmt
 - processes required to manage the timely completion of the project

<img src="images/ScheduleMgt.png" alt="ScheduleMgt" height="400" width="300">

## 1. Plan Schedule 
establishing the policies, procedures, and documentation for planning, developing, managing, executing, and controlling the
project schedule. 

**Key Benefit**: provides guidance and direction on how the project schedule will be managed throughout the project.

**ITTO (Input, Tools & Techniques, Output)**
| Inputs                                      | Tools & Techniques                     | Outputs              |
|--------------------------------------------|----------------------------------------|-----------------------|
| 1. Project Charter                         | 1. Expert judgement                    | 1. Schedule Plan      |
| 2. Project Plan                            | 2. Data gathering                      |                       |
| 3. EEFs (Enterprise Environmental Factors) | 3. Meetings                            |                       |
| 4. OPAs (Organizational Process Assets)    |                                        |                       |

**Output:**
- Schedule Plan: Establishes the criteria and the activities for developing, monitoring, and controlling the schedule.

**Template:** [Schedule Plan](../../templates/ScheduleMgt_template.docx)

## 2. Define Activities
 - Process of identifying and documenting the specific actions to be performed to produce the project deliverables
<img src="images/ScheduleMgt.png" alt="ScheduleMgt" height="400" width="300">

**Key Benefit**: that it decomposes work packages into schedule activities that provide a basis for
estimating, scheduling, executing, monitoring, and controlling the project work

**ITTO (Input, Tools & Techniques, Output)**
| Inputs                                      | Tools & Techniques                     | Outputs              |
|--------------------------------------------|----------------------------------------|-----------------------|
| 1. Project Charter                         | 1. Expert judgement                    | 1. Activity list      |
| 2. Project Plan                            | 2. Decomposition                      | 2. Milestone list      |
| 3. EEFs (Enterprise Environmental Factors) | 3. Rolling wave planning              |                       |
| 4. OPAs (Organizational Process Assets)    |                                        |                       |

**Output:**
- Activity list: Schedule activities required on the project
- Milestone list: milestone is a significant point or event in a project

## 3. Sequence Activities
 - Process of identifying and documenting relationships among the project activities

**Key Benefit**: Defines the logical sequence of work

**ITTO (Input, Tools & Techniques, Output)**
| Inputs                                      | Tools & Techniques                     | Outputs              |
|--------------------------------------------|----------------------------------------|-----------------------|
| 1. Project plan                           | 1. Precedence diagramming                    | 1. Project schedule network      |
| 2. Project doc                            | 2. Dependency diagramming                      | 2. Prod doc updates      |
| 3. EEFs (Enterprise Environmental Factors) | 3. Leads and Lags              |                       |
| 4. OPAs (Organizational Process Assets)    |                                        |                       |

**Output:**
Project schedule network diagrams show the order in which activities should be scheduled to address logical relationships between these activities
typically comprises of nodes that represent activities and arrows that show the sequence and dependencies

2.Dependency Determination: 
Mandatory: Ex. Foundation of the house erected prior to the house being built
Discretionary dependency: Ex. Painting the walls of a room & laying carpet at the same time
External dependency: Ex. The gas station receiving Gas prior to you filling up the Bulldozer gas tank
Internal dependency: PC testing cannot commence without completion of assembly and OS loading

 - A lead is the amount of time a successor activity can be advanced with respect to a predecessor activity.
 - A lag is the amount of time a successor activity will be -delayed with respect to a predecessor activity.
