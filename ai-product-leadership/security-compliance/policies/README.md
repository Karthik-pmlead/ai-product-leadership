# Security & Compliance Policies

This folder (`security-compliance/policies/`) contains **high‑level policy documents** aligned with:
- GDPR‑style data‑protection,
- ISO‑27001‑style information‑security,
- CE‑Marking‑style product‑safety thinking,
and OWASP‑style secure‑coding expectations.

These policies are written for **product leaders, engineering teams, and operations** to understand baseline security, privacy, and safety expectations for the products and systems we ship.

---

## Files in this folder

| File name                           | Purpose (what it covers) |
|-------------------------------------|---------------------------|
| [`data-protection-policy.md`][dpp]         | How we handle personal data, consent, and GDPR‑aligned expectations. |
| [`security-policy.md`][sp]                | Overall security expectations for systems, data, access, and code. |
| [`access-control-policy.md`][acp]          | How we define and manage roles, privileges, and admin‑access. |
| [`secure-development-policy.md`][sdp]      | How security and compliance are baked into the SDLC and product‑design. |
