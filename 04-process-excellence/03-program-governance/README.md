# 🚀 Governance Components

## 1. RAID Management

Track:
- Risks
- Assumptions
- Issues
- Dependencies

### Governance Rules
- Every risk assigned an owner
- Severity scoring mandatory
- Mitigation plan required
- Weekly status review

---

## 2. Intake & Scope Governance

### Intake Requirements
- Business objective
- OKR linkage
- ROI justification
- Resource estimate
- Risk assessment

### Scope Control Rules
- All scope changes require approval
- High-risk initiatives reviewed by leadership
- Dependencies documented before execution

---

## 3. Dependency Management

| Dependency Type | Example |
|---|---|
| Engineering | Shared services |
| Data | Pipeline readiness |
| Vendor | API integrations |
| Product | Requirement alignment |

### Coordination Mechanisms
- Weekly dependency review
- Escalation pathways
- Shared delivery dashboards
