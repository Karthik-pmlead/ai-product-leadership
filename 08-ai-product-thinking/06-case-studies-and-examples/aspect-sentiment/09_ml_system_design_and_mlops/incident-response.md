# Incident Response

## Purpose
Handle production problems safely.

## Incident types
- broken data feed.
- wrong output pattern.
- confidence collapse.
- taxonomy mismatch.
- review backlog overload.

## Response
1. detect issue.
2. pause or degrade safely.
3. notify owner.
4. fix root cause.
5. re-evaluate before resuming.

## Principle
Protect trust first, then restore speed.
