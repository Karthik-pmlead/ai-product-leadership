# Privacy and Security

## Purpose
Protect customer data while processing feedback.

## Controls

| Control | Description |
|---|---|
| Access control | Only approved users can access raw feedback. |
| Masking | Remove or hide sensitive fields when possible. |
| Retention | Keep only what is needed for the approved use case. |
| Audit log | Record access and changes to data. |
| Secure storage | Store data in approved systems only. |

## Security note
Customer feedback can contain personal or sensitive information, so the system should minimize exposure and follow internal policy.
