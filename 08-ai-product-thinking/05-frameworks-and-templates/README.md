# 05 — Frameworks and Templates

This section contains reusable AI product, system design, evaluation, and execution templates used across all projects in this repository.

It acts as the operating toolkit for building, evaluating, and launching AI-powered systems.

---

# 🎯 Purpose

To standardize how AI products are:
- defined (PRDs)
- evaluated (metrics + experiments)
- designed (system architecture)
- governed (responsible AI)
- launched (execution checklists)

---

# 🧠 Why This Exists

Most AI projects fail not because of code quality, but because of:
- unclear problem framing
- weak evaluation strategy
- missing governance
- inconsistent execution

These templates solve that by enforcing structured thinking before implementation.

---

# 📦 What’s Inside

| Template | Purpose |
|---|---|
| ai-prd-canvas-template.md | Structured PRD for AI products |
| ai-decision-matrix-template.md | Feature / model / idea prioritization |
| business-case-template.md | Executive-level justification |
| solution-design-template.md | System + AI architecture design |
| eval-plan-template.md | Offline + online evaluation strategy |
| model-card-template.md | Model documentation & transparency |
| mlops-checklist-template.md | Production readiness checklist |
| launch-checklist-template.md | Go-live execution checklist |

---

# 🧩 How These Fit Together

Idea → PRD → Decision Matrix → System Design → Evaluation Plan → Model Card → Launch Checklist

Each template supports one stage of the AI product lifecycle.

---

# 🧠 Key Design Principles

## 1. AI-First Thinking
Every template includes:
- model behavior considerations
- evaluation strategy
- failure modes
- explainability needs

---

## 2. Enterprise Readiness
Templates are designed for:
- fintech
- SaaS platforms
- AI product teams
- consulting-grade systems

---

## 3. Execution-Oriented
Each template is actionable, not theoretical:
- clear inputs
- structured outputs
- decision checkpoints

---

# 🚀 How to Use

## Step 1: Start with PRD Canvas
Define:
- problem
- users
- goals
- constraints

## Step 2: Use Decision Matrix
Prioritize:
- features
- models
- experiments

## Step 3: System Design
Define:
- architecture
- data flow
- AI components

## Step 4: Evaluation Plan
Define:
- offline metrics
- online A/B tests
- success thresholds

## Step 5: Model Card
Document:
- model behavior
- limitations
- risks
- fairness considerations

## Step 6: Launch Checklist
Ensure:
- observability
- rollback plan
- monitoring
- readiness checks

---

# 📊 Example Flow

For a Retail Recommendation System:

PRD → “Personalized product ranking”
↓
Decision Matrix → Rank session vs collaborative filtering
↓
System Design → Real-time ranking pipeline
↓
Eval Plan → CTR, conversion uplift
↓
Model Card → Bias, cold start issues
↓
Launch Checklist → A/B rollout + monitoring
