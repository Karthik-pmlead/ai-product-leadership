# Implementation and Roadmap

## Phases
1. define scope.
2. collect sample items.
3. define taxonomy.
4. build prototype.
5. validate on test cases.
6. launch pilot.
7. improve and expand.

## Milestones
- intake stream selected.
- routing taxonomy approved.
- prototype tested.
- pilot launched.
- evaluation complete.
- governance review passed.

## Dependencies
- historical examples.
- routing owners.
- CRM or workflow integration.
- human review support.
- stakeholder approval.

## Launch plan
Start with one sales intake stream and one routing objective.

## Adoption plan
Place triage into the existing sales or operations workflow so the team gets value without switching tools.
