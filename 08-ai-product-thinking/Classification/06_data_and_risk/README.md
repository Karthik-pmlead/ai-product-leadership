
# Data and Risk

## Data sources
- inbound emails.
- lead forms.
- CRM notes.
- support-like sales requests.
- call summaries.
- chat messages.
- metadata such as source, region, product, or segment.

## Data needs
- historical triage labels.
- routing outcomes.
- category definitions.
- review decisions.
- item metadata.

## Key risks
- unclear taxonomy.
- inconsistent labels.
- noisy short text.
- routing bias.
- data privacy issues.
- overconfidence in predictions.

## Mitigations
- define a small stable taxonomy.
- review label quality.
- use confidence thresholds.
- keep human fallback.
- protect customer and deal data.

## Risk posture
The first release should focus on one sales intake stream and one routing objective.
