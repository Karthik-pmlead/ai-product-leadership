# Information Architecture

# AI-Powered Investment Banking Workspace

---

# Purpose

The information architecture defines how users discover, navigate, and interact with information across the Investment Banking Workspace.

The objective is to minimize cognitive load by organizing data around user workflows rather than underlying systems.

---

# Design Principles

- Workflow-first navigation
- Search before browse
- AI-assisted discovery
- Progressive disclosure
- Consistent terminology
- Role-aware experiences

# Primary Objects

Users interact with several core business objects.

- Company
- Client
- Deal
- Executive
- Research Report
- Financial Statement
- Meeting
- Task
- Opportunity

Each object provides a consistent layout, actions, and navigation.

---

# Content Organization

Structured Data

- Financial metrics
- Market data
- Transactions
- CRM records

Unstructured Data

- Research reports
- Meeting notes
- Presentations
- Emails
- Regulatory filings

AI surfaces both seamlessly through unified search.

# Search Integration

- Natural language queries
- Semantic search
- Filters
- Recent activity
- Saved searches

---

# Personalization

Users see:

- Recent companies
- Active deals
- Saved searches
- Recommended actions
- Personalized watchlists

---

# Product Takeaway

The information architecture is designed around how bankers think and work, enabling them to move naturally between research, relationships, deals, and AI-assisted insights without navigating disconnected systems.
