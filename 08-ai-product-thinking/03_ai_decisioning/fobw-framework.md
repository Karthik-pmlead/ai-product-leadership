# FOBW Framework

## Purpose
FOBW is the lens used to decide whether an AI solution is the right path for a business problem.

## Framework

| Letter | Meaning | Questions to ask |
|---|---|---|
| F | Fit | Is this a good problem for AI? |
| O | Outcome | What measurable business outcome do we want? |
| B | Business value | Does solving this create meaningful value? |
| W | Way to solve | What is the simplest workable solution? |

## Application to aspect sentiment

| FOBW step | Answer |
|---|---|
| Fit | Yes, because the input is unstructured text and the output is repeated classification. |
| Outcome | Better customer insight, faster prioritization, and stronger CX decisions. |
| Business value | High, because teams can act on frequent negative themes. |
| Way to solve | Start with a hybrid system, then refine based on pilot results. |

## Decision rule
If the problem is text-heavy, repeated, measurable, and business-critical, FOBW will usually support an AI approach. If the task can be solved well by a simple rule or existing process, do not overcomplicate it.

## Output
Use FOBW as a gate before committing to the PRD and solution design stages.
