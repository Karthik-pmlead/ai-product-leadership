# RACI Matrix

## Overview

RACI is a responsibility assignment framework used to clarify:
- ownership
- accountability
- collaboration
- decision-making

It helps reduce confusion across teams by defining who is responsible for what.

---

# RACI Meaning

| Letter | Meaning | Description |
|---|---|---|
| R | Responsible | Executes the work |
| A | Accountable | Ultimately owns the outcome |
| C | Consulted | Provides input/advice |
| I | Informed | Kept updated |

---

# Core Principle

Every task should ideally have:
- one clear Accountable owner

Too many accountable owners create ambiguity.

---

# Example

| Task | PM | Engineer | Designer | QA |
|---|---|---|---|---|
| Define Requirements | A | C | C | I |
| Build Feature | I | R | I | I |
| UX Design | C | I | R | I |
| Testing | I | C | I | R |

---

# Best Used When

- Cross-functional projects
- Enterprise programs
- Product launches
- Governance processes
- Team coordination

---

# Advantages

- Clarifies ownership
- Reduces duplication
- Improves communication
- Prevents decision confusion

---

# Limitations

- Can become overly bureaucratic
- Too much granularity reduces usefulness
- Doesn't replace execution management

---

# Best Practices

- Keep matrices lightweight
- Assign one accountable owner
- Review regularly
- Align with team workflows
