# DMAIC Framework

## Overview

DMAIC is a structured problem-solving and process improvement framework used in:
- Six Sigma
- operational excellence
- quality improvement

DMAIC stands for:
- Define
- Measure
- Analyze
- Improve
- Control

---

# DMAIC Phases

| Phase | Purpose |
|---|---|
| Define | Clarify problem & goals |
| Measure | Collect performance data |
| Analyze | Identify root causes |
| Improve | Implement solutions |
| Control | Sustain improvements |

---

# Process Flow

```text
Define → Measure → Analyze → Improve → Control
```

---

# Phase Breakdown

## Define

Clarify:
- problem statement
- goals
- stakeholders
- scope

Example:
> Checkout abandonment increased by 18%.

---

## Measure

Collect baseline data.

Examples:
- conversion rates
- latency
- defect rates

---

## Analyze

Identify root causes using:
- 5 Whys
- Fishbone diagrams
- data analysis

---

## Improve

Implement process or product improvements.

Examples:
- UI redesign
- automation
- workflow optimization

---

## Control

Prevent regression through:
- monitoring
- documentation
- process controls

---

# Best Used When

- Operational improvement
- Quality management
- Process optimization
- Incident reduction

---

# Advantages

- Structured methodology
- Data-driven improvement
- Strong root-cause focus
- Sustainable optimization

---

# Limitations

- Can become process-heavy
- Slower than lightweight experimentation
- Requires quality data
