# 🧠 5 Whys Framework

## 📌 What is it?

**5 Whys** is a root cause analysis technique used to identify the underlying cause of a problem by repeatedly asking “Why?”

It helps PMs move from **symptoms → real problem**

---

## ⏰ When to Use 5 Whys

Use 5 Whys when:

- A problem keeps recurring
- You need to find **root cause, not symptoms**
- There is a **product bug or failure**
- You are analyzing **user drop-offs**
- You want to improve system reliability

👉 Best for: **problem-solving + debugging + product issues**

---

## 📍 Where to Use It

- Product incident analysis  
- UX problem investigation  
- Customer support issues  
- Sprint retrospectives  
- Process improvement  

---

## ⚙️ How 5 Whys Works

Ask “Why?” repeatedly until you reach the root cause (usually 5 times, but can be fewer or more).

---

## 🍔 Example (Food Delivery App)

**Problem:** Users are abandoning checkout

| Step | Question | Answer |
|------|----------|--------|
| 1 | Why are users abandoning checkout? | Payment fails |
| 2 | Why does payment fail? | OTP is not received |
| 3 | Why is OTP not received? | SMS gateway is delayed |
| 4 | Why is SMS gateway delayed? | High traffic load |
| 5 | Why is there no backup system? | No fallback provider integrated |

👉 Root cause: No redundant SMS gateway system

---

## 🧠 Simple Mental Model

- Keep asking “Why?”
- Don’t stop at surface level
- Go until you reach system-level cause

---

## 🚀 PM Insight

> Most product problems are not feature problems — they are system problems.

5 Whys helps uncover that.


# 💡 1H Framework (How Thinking)

## 📌 What is it?

The **1H (How)** framework focuses on generating solutions by asking:

> “How can we solve this problem?”

It is a **solution-oriented thinking tool** used in early ideation.

---

## ⏰ When to Use 1H

Use 1H when:

- You already understand the problem
- You need to generate **solution ideas**
- You are in brainstorming phase
- You want to explore **multiple solution paths**
- You are designing new features

👉 Best for: **ideation + solution generation**

---

## 📍 Where to Use It

- Product brainstorming sessions  
- UX workshops  
- Feature design discussions  
- Startup idea exploration  
- Design thinking sessions  

---

## ⚙️ How 1H Works

Start with a problem statement, then ask:

> “How might we solve this?”

Then generate multiple solution directions.

---

## 🍔 Example (Food Delivery App)

**Problem:** Users abandon checkout due to long payment flow

### 💡 How ideas:

- How can we reduce checkout steps?
- How can we make payments faster?
- How can we enable 1-click ordering?
- How can we pre-save payment methods?
- How can we reduce friction in OTP verification?

---

## 🧠 Simple Mental Model

- Start with problem
- Ask “How?”
- Generate multiple solutions
- Do not judge ideas early

---

## 🚀 PM Insight

> 5 Whys finds the problem. 1H creates solutions.

Together, they form a **problem → solution thinking loop**.
