# HEART Framework

## Overview

The HEART Framework is a UX metrics framework developed by:
- Google UX

It helps teams measure user experience across five dimensions.

HEART stands for:
- Happiness
- Engagement
- Adoption
- Retention
- Task Success

---

# The 5 Dimensions

| Dimension | Meaning |
|---|---|
| Happiness | User satisfaction |
| Engagement | Level of interaction |
| Adoption | New user uptake |
| Retention | Continued usage |
| Task Success | Ability to complete tasks |

---

# Dimension Breakdown

## Happiness

Measures:
- satisfaction
- perceived usability
- sentiment

### Metrics
- CSAT
- NPS
- App ratings

---

## Engagement

Measures:
- depth/frequency of interaction

### Metrics
- Session duration
- Feature usage
- DAU/MAU

---

## Adoption

Measures:
- new user/product adoption

### Metrics
- New signups
- Feature adoption rate

---

## Retention

Measures:
- returning users

### Metrics
- Churn
- Retention cohorts

---

## Task Success

Measures:
- effectiveness and efficiency

### Metrics
- Completion rate
- Error rate
- Time on task

---

# Best Used When

- UX measurement
- Product analytics
- Feature evaluation
- Customer experience tracking

---

# Advantages

- Balances UX + business metrics
- Encourages holistic measurement
- Flexible across products

---

# Limitations

- Requires good instrumentation
- Metrics selection can become subjective

---
