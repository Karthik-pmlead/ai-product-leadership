# 🎯 PRD Evaluation Framework (SIMPLE-FAV)

## 📌 Acronym: SIMPLE-FAV

A simple way to evaluate any product or PRD using core UX principles.

---

## 🧠 Meaning

### 🟢 SIMPLE
| Letter | Meaning | What it checks |
|--------|--------|----------------|
| S | Simple | Is it easy to use? |
| I | Intuitive | Can users understand it instantly? |
| M | Mapping | Does it match real-world mental models? |
| P | Predictable | Does it behave consistently? |
| L | Logical | Is the flow structured and sensible? |
| E | Efficient | Can users complete tasks quickly? |

---

### ⭐ FAV (Experience Quality)
| Letter | Meaning | What it checks |
|--------|--------|----------------|
| F | Feedback | Does system respond clearly to actions? |
| A | Affordance | Do UI elements suggest how to use them? |
| V | Visibility | Is system status clear and visible? |

---

## ⏰ When to Use It

Use SIMPLE-FAV when:

- Reviewing a **PRD or feature design**
- Doing a **UX critique**
- Comparing **design alternatives**
- Preparing for **PM interviews**
- Auditing product experience

---

## 📍 Where to Use It

- PRD reviews  
- UX design critiques  
- Product teardown sessions  
- Interview preparation  
- Feature validation  

---

## ⚙️ How to Use It

### Step 1: Pick a feature
Example: Checkout flow in a food delivery app

### Step 2: Run SIMPLE-FAV check

Ask:

- Is it SIMPLE to use?
- Is it INTUITIVE?
- Does it follow MAPPING?
- Is it PREDICTABLE?
- Is it LOGICAL?
- Is it EFFICIENT?
- Does it give FEEDBACK?
- Does it show AFFORDANCE?
- Is system VISIBILITY clear?

### Step 3: Mark gaps
- ✔️ good
- ⚠️ needs improvement
- ❌ missing

---

## 🍔 Example (Food Delivery App Checkout)

| Principle | Evaluation |
|----------|------------|
| Simple | Too many steps ❌ |
| Intuitive | Moderate ⚠️ |
| Mapping | Confusing order flow ⚠️ |
| Predictable | Good ✔️ |
| Logical | Good ✔️ |
| Efficient | Slow due to OTP delays ❌ |
| Feedback | Strong ✔️ |
| Affordance | Good ✔️ |
| Visibility | Strong ✔️ |

---

## 🧠 Easy Memory Trick

👉 Think:

> SIMPLE = How easy it is  
> FAV = How good it feels  

---

## 🚀 PM Insight

> Great products are not feature-rich — they are **simple to understand and effortless to use**

SIMPLE-FAV helps you quickly spot:
- UX friction
- Missing clarity
- Poor interaction design

# 🎯 SIMPLE + FAV Evaluation (Don Norman Example)

## 📱 Product Example: ATM Machine

This evaluates ATM using UX principles inspired by **Don Norman (Design of Everyday Things)**.

---

## 🧠 SIMPLE + FAV Evaluation

| Principle | Meaning (Don Norman mapping) | ATM Example Evaluation |
|----------|------------------------------|------------------------|
| S – Simple | Minimal cognitive load | Only essential options shown (Withdraw, Balance, Deposit) ✔️ |
| I – Intuitive | Easy to understand without instructions | Users naturally follow: Insert card → Enter PIN → Choose action ✔️ |
| M – Mapping | Matches real-world mental model | Buttons directly map to cash actions ✔️ |
| P – Predictable | Consistent behavior across usage | Same flow across all ATMs ✔️ |
| L – Logical | Steps follow natural sequence | Card → PIN → Select → Cash ✔️ |
| E – Efficient | Fast task completion | Withdrawal typically under 1 minute ✔️ |
| F – Feedback | System responds clearly | “Transaction successful” + cash dispensed ✔️ |
| A – Affordance | UI indicates how to use it | Card slot clearly suggests “insert card here” ✔️ |
| V – Visibility | System status is clear | Shows balance, errors, and step progress ✔️ |

---

## 🧠 Don Norman Insight

ATM is a classic example of good design because it:
- Reduces memory load (recognition over recall)
- Provides immediate feedback for every action
- Uses strong real-world mapping (cash → actions)
- Prevents errors through constraints (invalid PIN lockout)
- Keeps system status visible at all times

---

## 🚀 PM Takeaway

> A well-designed system makes users feel like “nothing can go wrong.”

ATM works well because:
- Users are never confused
- Every step is visible and predictable
- Errors are prevented, not just corrected

---

## 🧩 One-line Summary

👉 **ATM = Simple flow + strong mapping + clear feedback + visible system state (perfect Don Norman UX)**
