# Design Thinking

## Overview

Design Thinking is a human-centered problem-solving framework focused on:
- empathy
- experimentation
- iteration
- creativity

It helps teams solve complex problems by deeply understanding users before designing solutions.

Popularized by:
- IDEO
- Stanford d.school

---

# The 5 Stages

| Stage | Purpose |
|---|---|
| Empathize | Understand users |
| Define | Frame the problem |
| Ideate | Generate ideas |
| Prototype | Build testable solutions |
| Test | Validate with users |

---

# Process Flow

```text
Empathize → Define → Ideate → Prototype → Test
```

The process is iterative, not strictly linear.

---

# Stage Breakdown

## Empathize

Research users through:
- interviews
- observation
- immersion
- journey mapping

Goal:
Understand real user needs.

---

## Define

Synthesize insights into:
- clear problem statements
- user needs
- opportunity areas

Example:
> Users struggle to complete onboarding without guidance.

---

## Ideate

Generate many solution ideas using:
- brainstorming
- SCAMPER
- Crazy 8s
- How Might We

---

## Prototype

Create lightweight solutions:
- wireframes
- mockups
- clickable prototypes
- MVPs

---

## Test

Validate solutions with users:
- usability testing
- interviews
- experiments
- A/B testing

---

# Core Principles

| Principle | Meaning |
|---|---|
| Human-Centered | Focus on user needs |
| Iterative | Continuously refine |
| Collaborative | Cross-functional participation |
| Experimental | Learn through testing |

---

# Best Used When

- Product discovery
- UX design
- Innovation workshops
- Complex ambiguous problems
- Customer experience redesign

---

# Advantages

- Deep customer empathy
- Encourages innovation
- Reduces solution bias
- Supports iterative learning

---

# Limitations

- Time-intensive
- Requires strong facilitation
- Can become overly workshop-heavy

---

# Example

## Food Delivery App

### Problem
Users abandon checkout.

### Discovery
Users feel checkout takes too long.

### Solution
One-click checkout prototype.

### Test Result
Checkout completion improves by 18%.

---

# Best Practices

- Start with customer understanding
- Prototype early
- Test frequently
- Embrace iteration
- Encourage divergent thinking
