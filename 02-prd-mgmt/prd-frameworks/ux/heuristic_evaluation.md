# Heuristic Evaluation

## Overview

Heuristic Evaluation is a usability inspection method where experts evaluate a product interface against established usability principles (“heuristics”).

Popularized by:
- Jakob Nielsen

The goal is to identify:
- usability issues
- UX friction
- interaction inconsistencies

---

# Nielsen’s 10 Usability Heuristics

| Heuristic | Description |
|---|---|
| Visibility of System Status | Keep users informed |
| Match Between System & Real World | Use familiar language |
| User Control & Freedom | Easy undo/exit |
| Consistency & Standards | Predictable interactions |
| Error Prevention | Prevent mistakes |
| Recognition Over Recall | Reduce memory load |
| Flexibility & Efficiency | Support novice + expert users |
| Aesthetic & Minimal Design | Avoid clutter |
| Help Users Recover from Errors | Clear recovery guidance |
| Help & Documentation | Accessible support |

---

# Evaluation Process

1. Define evaluation scope
2. Review interface screens/flows
3. Identify heuristic violations
4. Assign severity ratings
5. Recommend improvements

---

# Severity Ratings

| Severity | Meaning |
|---|---|
| 0 | No issue |
| 1 | Cosmetic issue |
| 2 | Minor usability issue |
| 3 | Major usability issue |
| 4 | Critical usability issue |

---

# Example

## Issue
Users cannot easily find “Cancel Subscription.”

### Violated Heuristic
User Control & Freedom

### Severity
3 — Major usability issue

---

# Best Used When

- UX reviews
- Product audits
- Prototype evaluation
- Design QA
- Accessibility improvements

---

# Advantages

- Fast and low-cost
- Identifies common usability problems
- Useful early in design process
- Doesn't require large user studies

---

# Limitations

- Expert-dependent
- May miss real-world behaviors
- Doesn't replace user testing

---

# Best Practices

- Use multiple evaluators
- Combine with usability testing
- Prioritize high-severity issues
- Evaluate real user flows
