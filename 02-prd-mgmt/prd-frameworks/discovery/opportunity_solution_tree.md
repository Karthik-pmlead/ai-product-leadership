# Opportunity Solution Tree (OST)

## Overview

Opportunity Solution Tree (OST) is a continuous product discovery framework created by:
- Teresa Torres

It helps teams connect:
- outcomes
- opportunities
- solutions
- experiments

The framework ensures teams solve meaningful customer problems instead of jumping directly to features.

---

# Core Structure

```text
Outcome
 ├── Opportunities
 │    ├── Solutions
 │    │    ├── Experiments
```

---

# Components

| Component | Description |
|---|---|
| Outcome | Desired business/customer result |
| Opportunity | Customer pain point or unmet need |
| Solution | Possible approach |
| Experiment | Validate assumptions |

---

# Example

## Outcome
Increase user retention.

### Opportunity
Users abandon onboarding.

#### Solutions
- Guided onboarding
- Personalized checklist
- Progress tracker

##### Experiments
- A/B test onboarding flow
- User interviews

---

# Best Used When

- Continuous discovery
- Product strategy
- Feature prioritization
- Discovery roadmaps
- Experimentation planning

---

# Advantages

- Customer-centric
- Prevents feature factory mindset
- Encourages experimentation
- Aligns strategy with execution

---

# Limitations

- Requires strong discovery culture
- Needs continuous customer research
- Can become large and complex

---

# Best Practices

- Start with measurable outcomes
- Validate opportunities with research
- Test assumptions early
- Keep trees iterative and evolving
