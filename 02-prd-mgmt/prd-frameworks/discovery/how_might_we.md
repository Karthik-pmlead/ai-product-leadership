# How Might We (HMW)

## Overview

How Might We (HMW) is a problem reframing technique used in design thinking and product discovery.

It transforms problems into opportunity-oriented questions.

Instead of:
- focusing on constraints

It encourages:
- possibility thinking
- collaboration
- ideation

---

# Structure

Typical format:

```text
How might we [goal] for [user] in a way that [outcome]?
```

---

# Examples

## Weak Statement
Users hate onboarding.

## Better HMW
How might we make onboarding feel effortless for first-time users?

---

## Another Example
How might we reduce checkout friction for mobile users?

---

# Best Used When

- Discovery workshops
- Product ideation
- UX brainstorming
- Innovation sessions
- Problem reframing

---

# Characteristics of Good HMW Questions

| Good HMW | Poor HMW |
|---|---|
| Open-ended | Too narrow |
| Human-centered | Solution-biased |
| Inspires ideas | Restrictive |
| Actionable | Vague |

---

# Advantages

- Encourages creativity
- Reframes negativity
- Improves collaboration
- Expands solution space

---

# Limitations

- Poor wording reduces effectiveness
- Too broad can become unfocused
- Requires facilitation

---

# Example Workflow

1. Identify user pain point
2. Convert into HMW statement
3. Brainstorm solutions
4. Prioritize ideas
5. Prototype/test
