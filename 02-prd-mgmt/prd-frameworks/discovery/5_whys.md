# 5 Whys

## Overview

5 Whys is a root cause analysis technique used to identify the underlying cause of a problem by repeatedly asking:

> “Why did this happen?”

The goal is to move beyond symptoms and uncover systemic causes.

Originally developed within:
- Toyota Production System

---

# Core Idea

Each answer becomes the basis for the next “Why?”

Typically:
- 5 iterations are enough
- but the exact number may vary

---

# Process

1. Define the problem
2. Ask “Why?” the problem occurred
3. Continue asking “Why?” for each answer
4. Identify the root cause
5. Define corrective actions

---

# Example

## Problem
Customers abandon checkout.

### Why 1
Checkout takes too long.

### Why 2
Pages load slowly.

### Why 3
API response time is high.

### Why 4
Database queries are inefficient.

### Why 5
Indexes were never optimized.

## Root Cause
Poor database optimization.

---

# Best Used When

- Incident analysis
- Operational issues
- Product failures
- Customer complaints
- Process improvement

---

# Advantages

- Simple and fast
- Encourages deeper thinking
- Identifies systemic issues
- No special tools required

---

# Limitations

- Can become subjective
- Depends on facilitator quality
- May oversimplify complex systems
- Multiple root causes may exist

---

# Best Practices

- Focus on facts, not blame
- Use collaborative discussion
- Validate assumptions with data
- Stop when actionable root causes emerge
