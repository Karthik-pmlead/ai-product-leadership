# Wardley Mapping

## Overview

Wardley Mapping is a strategic mapping framework created by:
- Simon Wardley

It visualizes:
- user needs
- business capabilities
- technology evolution

The goal is to improve strategic decision-making.

---

# Core Dimensions

| Dimension | Meaning |
|---|---|
| Value Chain | User-facing to invisible systems |
| Evolution | Genesis → Commodity |

---

# Evolution Stages

| Stage | Description |
|---|---|
| Genesis | Novel/experimental |
| Custom-Built | Emerging solutions |
| Product | Standardized offerings |
| Commodity | Utility-like services |

---

# Core Idea

Not everything should be built internally.

Commodity components may be:
- outsourced
- automated
- standardized

---

# Best Used When

- Technology strategy
- Platform planning
- Cloud transformation
- Competitive analysis

---

# Advantages

- Encourages strategic clarity
- Visualizes ecosystem evolution
- Helps investment decisions

---

# Limitations

- Steep learning curve
- Requires strategic understanding
- Subjective mapping possible

---

# Example

A company maps:
- customer app
- authentication
- cloud infrastructure

Then identifies:
- which capabilities differentiate
- which should commoditize
