# DORA Metrics

## Overview

DORA Metrics are a set of engineering performance metrics created by:
- DevOps Research and Assessment (DORA)

They measure:
- software delivery performance
- operational excellence
- engineering effectiveness

Widely adopted in:
- DevOps
- platform engineering
- SRE organizations

---

# The 4 Core Metrics

| Metric | Description |
|---|---|
| Deployment Frequency | How often code is deployed |
| Lead Time for Changes | Time from commit → production |
| Change Failure Rate | % of deployments causing failures |
| Mean Time to Recovery (MTTR) | Time to recover from incidents |

---

# Metric Breakdown

## Deployment Frequency

Measures:
- release velocity

Higher frequency often indicates:
- strong automation
- smaller deployments
- agile delivery

---

## Lead Time for Changes

Measures:
- engineering delivery speed

Tracks:
```text
Code Commit → Production Release
```

---

## Change Failure Rate

Measures:
- release reliability

Examples:
- outages
- rollback events
- hotfixes

---

## MTTR (Mean Time to Recovery)

Measures:
- operational resilience

Tracks:
```text
Incident Start → Service Recovery
```

# Performance Levels

| Level | Characteristics |
|---|---|
| Elite | Fast + reliable delivery |
| High | Strong engineering maturity |
| Medium | Moderate operational efficiency |
| Low | Slow & unstable delivery |

# Best Used When

- Engineering performance measurement

---

# Advantages

- Outcome-focused metrics
- Encourages operational excellence
- Strong industry benchmarking
- Balances speed + stability

---

# Limitations

- Metrics alone don't capture engineering quality
- Can be gamed if misused
- Context matters across organizations

---

# Best Practices

- Optimize flow + reliability together
- Avoid vanity metric pressure
- Automate deployments & testing
- Pair with developer experience metrics
