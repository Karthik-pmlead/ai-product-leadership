# Definition of Done (DoD)

## Overview

Definition of Done (DoD) is a shared agreement that defines when work is considered fully complete.

It ensures:
- quality consistency
- delivery alignment
- shared expectations

A strong DoD reduces ambiguity.

---

# Core Idea

“Done” means:
- releasable quality
not merely:
- coding completed

---

# Common DoD Criteria

| Area | Example |
|---|---|
| Code Quality | Peer reviewed |
| Testing | Automated tests passed |
| Documentation | Updated |
| UX Validation | Approved |
| Deployment | Successfully deployed |
| Security | Vulnerability checks passed |

---

# Example DoD

A feature is “Done” when:
- code is merged
- tests pass
- QA approves
- analytics added
- documentation updated
- deployed to production

---

# Best Used When

- Agile teams
- Scrum environments
- Software delivery
- Quality management

---

# Advantages

- Shared expectations
- Improved quality
- Reduced rework
- Better release readiness

---

# Limitations

- Overly rigid DoDs slow teams
- Poorly defined criteria create confusion

---

# Best Practices

- Define collaboratively
- Keep criteria measurable
- Review periodically
- Align with quality standards
