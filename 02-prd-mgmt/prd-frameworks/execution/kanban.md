# Kanban

## Overview

Kanban is a workflow management framework focused on:
- continuous delivery
- visual workflow management
- limiting work in progress (WIP)

Originated from:
- Toyota Production System

---

# Core Principles

| Principle | Description |
|---|---|
| Visualize Work | Make workflow visible |
| Limit WIP | Reduce multitasking |
| Manage Flow | Optimize throughput |
| Continuous Improvement | Improve incrementally |

---

# Typical Workflow

```text
Backlog → In Progress → Review → Done
```

---

# Key Concepts

## Work In Progress (WIP) Limits

Restrict active tasks to avoid overload.

Example:
- Max 3 items in “In Progress”

---

## Flow Efficiency

Measure how smoothly work moves through the system.

---

## Cycle Time

Time taken from:
- work start
to
- completion

---

# Best Used When

- Operational teams
- Continuous delivery environments
- Support workflows
- DevOps processes

---

# Advantages

- Flexible prioritization
- Reduces bottlenecks
- Improves visibility
- Encourages continuous flow

---

# Limitations

- Less predictable cadence
- Weak prioritization may create chaos
- Requires discipline with WIP limits
