# A/B Testing

## Overview

A/B Testing is an experimentation method used to compare two or more variations to determine which performs better against a defined metric.

It is commonly used in:
- product optimization
- UX improvements
- marketing experiments
- growth initiatives

---

# Core Idea

Users are split into groups:

| Group | Experience |
|---|---|
| Control (A) | Existing version |
| Variant (B) | Modified version |

Performance differences are measured statistically.

---

# Experiment Flow

```text
Hypothesis
    ↓
Create Variants
    ↓
Split Traffic
    ↓
Measure Results
    ↓
Analyze Significance
    ↓
Ship or Reject
```

---

# Common Metrics

| Metric | Example |
|---|---|
| Conversion Rate | Signup completion |
| CTR | Button clicks |
| Retention | Returning users |
| Revenue | Purchase value |
| Engagement | Session duration |

---

# Example

## Hypothesis
Changing CTA color increases conversions.

| Variant | Conversion Rate |
|---|---|
| A (Blue CTA) | 4.2% |
| B (Green CTA) | 5.1% |

Result:
- Variant B performs better.

---

# Best Used When

- UX optimization
- Funnel improvements
- Feature validation
- Marketing experiments
- Pricing tests

---

# Advantages

- Data-driven decisions
- Reduces opinion bias
- Enables incremental optimization
- Supports experimentation culture

---

# Limitations

- Requires sufficient traffic
- Poor experiment design misleads
- Local optimizations may hurt overall UX

---

# Best Practices

- Test one major variable at a time
- Define success metrics upfront
- Ensure statistical significance
- Avoid premature conclusions
