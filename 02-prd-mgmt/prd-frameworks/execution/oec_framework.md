# OEC Framework (Overall Evaluation Criterion)

## Overview

The OEC Framework defines the primary metric used to evaluate whether a product or experiment improves overall user and business outcomes.

Popularized by:
- Microsoft experimentation teams

The OEC acts as:
- the “north star” metric for experiments

---

# Core Idea

Optimization should focus on:
- long-term value
not just local metrics.

A good OEC balances:
- user experience
- engagement
- business impact

---

# Characteristics of a Good OEC

| Characteristic | Description |
|---|---|
| Measurable | Quantifiable |
| Sensitive | Detects meaningful changes |
| Holistic | Reflects overall value |
| Long-term oriented | Avoids short-term optimization traps |

---

# Example OECs

| Product Type | Example OEC |
|---|---|
| Marketplace | Successful transactions |
| Streaming Platform | Hours watched |
| SaaS Tool | Weekly active teams |
| E-Commerce | Revenue per active user |

---

# Experiment Example

## Hypothesis
Reducing signup steps increases activation.

### OEC
Weekly retained activated users.

Not just:
- signup completion

This avoids optimizing vanity metrics.

---

# Best Used When

- Experimentation programs
- Growth optimization
- Product analytics
- Feature evaluation

---

# Advantages

- Aligns teams on meaningful outcomes
- Prevents local optimization traps
- Encourages long-term thinking

---

# Limitations

- Difficult to define correctly
- One metric may miss nuance
- Requires strong analytics maturity

---

# Best Practices

- Combine leading + lagging indicators
- Avoid vanity metrics
- Align OEC with customer value
- Revisit periodically
