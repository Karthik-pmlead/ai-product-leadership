# Hypothesis Framework

## Overview

A Hypothesis Framework structures assumptions into testable statements.

It helps teams:
- validate ideas quickly
- reduce uncertainty
- run effective experiments
- make evidence-based decisions

Widely used in:
- product discovery
- experimentation
- growth
- Lean Startup practices

---

# Core Structure

```text
We believe that
[doing this]
for
[these users]
will achieve
[this outcome].

We'll know this is true when
[this metric changes].
```

---

# Example

```text
We believe that simplifying onboarding
for new mobile users
will increase activation rates.

We'll know this is true when onboarding completion improves from 45% to 60%.
```

---

# Components

| Component | Description |
|---|---|
| Assumption | What you believe |
| Audience | Target users |
| Change | Proposed action |
| Outcome | Expected impact |
| Metric | Validation criteria |

---

# Best Used When

- Product discovery
- Experimentation planning
- Feature validation
- Growth optimization

---

# Advantages

- Encourages measurable thinking
- Reduces assumption bias
- Improves experiment quality
- Supports rapid learning

---

# Limitations

- Poor metrics weaken validation
- Assumptions may remain hidden
- Complex systems may require multiple hypotheses

---

# Best Practices

- Make hypotheses measurable
- Focus on user outcomes
- Define success thresholds upfront
- Validate quickly
