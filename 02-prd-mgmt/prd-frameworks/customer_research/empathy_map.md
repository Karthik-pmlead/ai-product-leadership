# Empathy Map

## Overview

An Empathy Map is a collaborative visualization framework used to better understand users.

It captures:
- thoughts
- emotions
- behaviors
- motivations
- frustrations

The goal is to build empathy for users.

---

# Empathy Map Sections

| Section | Description |
|---|---|
| Says | What users verbally express |
| Thinks | Internal thoughts & concerns |
| Does | Observable actions |
| Feels | Emotional state |
| Pains | Frustrations & obstacles |
| Gains | Desired outcomes |

---

# Example

## Food Delivery User

| Area | Example |
|---|---|
| Says | “Delivery takes too long.” |
| Thinks | “Can I trust this app?” |
| Does | Compares multiple apps |
| Feels | Frustrated when delayed |
| Pain | Hidden fees |
| Gain | Fast predictable delivery |

---

# Best Used When

- UX workshops
- Discovery sessions
- Journey mapping
- Product design
- Customer interviews

---

# Advantages

- Builds customer empathy
- Aligns teams around users
- Encourages human-centered design

---

# Limitations

- Can become assumption-based
- Needs research validation
- Less useful without real user data

---

# Best Practices

- Use interview insights
- Include cross-functional teams
- Focus on behaviors/emotions
- Keep maps updated
