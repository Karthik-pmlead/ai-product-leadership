# Service Blueprint

## Overview

A Service Blueprint extends a customer journey map by visualizing:
- frontstage interactions
- backstage operations
- systems
- internal processes

It helps organizations understand how services are delivered end-to-end.

---

# Core Layers

| Layer | Description |
|---|---|
| Customer Actions | User interactions |
| Frontstage | Visible employee/system interactions |
| Backstage | Internal operational processes |
| Support Systems | Technology/process support |

---

# Key Concepts

## Line of Interaction
Where customers interact with the service.

## Line of Visibility
Separates visible and invisible operations.

## Line of Internal Interaction
Separates support functions from employees.

---

# Example

## Food Delivery App

| Layer | Example |
|---|---|
| Customer Action | Place order |
| Frontstage | App confirms order |
| Backstage | Restaurant receives request |
| Support System | Payment gateway |

# Best Used When

- Service design
- Operational optimization
- Cross-functional alignment
- Complex customer journeys

---

# Advantages

- Connects UX with operations
- Identifies operational bottlenecks
- Improves service delivery
- Helps system thinking

---

# Limitations

- Can become highly complex
- Requires cross-team collaboration
- Time-intensive workshops

---

# Best Practices

- Start with customer journey
- Include operational teams
- Keep blueprint readable
- Focus on critical flows first
