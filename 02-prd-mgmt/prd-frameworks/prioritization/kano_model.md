# Kano Model

## Overview

The Kano Model helps product teams categorize features based on how they influence customer satisfaction.

It distinguishes between:
- Basic expectations
- Performance improvements
- Delightful innovations

Created by:
- Professor Noriaki Kano

---

# Kano Categories

## Must-Have Features

Basic expectations users assume will exist.

If absent:
- Users become dissatisfied

If present:
- Satisfaction does not significantly increase

Examples:
- Login functionality
- Payment security

---

## Performance Features

More performance = more satisfaction.

Examples:
- Faster delivery
- Better battery life
- Improved search relevance

---

## Delighters

Unexpected features that create excitement.

Examples:
- AI recommendations
- Smart automation
- Surprise rewards

---

## Indifferent Features

Features users don't care about.

---

## Reverse Features

Features some users dislike.

Example:
- Excessive notifications

# Best Used When

- Feature prioritization
- Product roadmap planning
- Customer satisfaction analysis
- Innovation strategy

---

# Advantages

- Balances innovation vs fundamentals
- Prevents over-investment in basics
- Encourages customer-centric thinking

---

# Limitations

- Requires customer research
- Categories evolve over time
- Subjective interpretation possible

---

# Example

|---|---|
| Secure login | Must-have |
