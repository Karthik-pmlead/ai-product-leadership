## MoSCow

MoSCoW is a prioritization technique that splits work into 4 categories:

M → Must have (non-negotiable)
S → Should have (important but not critical). solution is still viable wihtout it
C → Could have (nice to have)
W → Won’t have (not now, not this release)

🧭 **When to Use MoSCoW**

- Use MoSCoW when:

- You have limited time or resources
- You are planning a release / sprint / MVP
-Stakeholders are asking for too many features
- You need clear scope boundaries
- You are building a roadmap or PRD

👉 Best fit: early product planning + MVP definition

📍 **Where to Use MoSCoW**

MoSCoW works best in:

📦 Product Requirement Documents (PRDs)
🗺️ Roadmaps
🏃 Sprint planning (Agile teams)
🚀 MVP definition
🤝 Stakeholder alignment meetings

⚙️ **How to Use MoSCoW**

Step 1: List all features / requirements
- Write everything down (no filtering yet)

Step 2: Categorize each item

Ask:
- Is this absolutely required to launch?
- Can we launch without this?
- Is this optional improvement?
- Should we explicitly NOT do this now?
  
Step 3: Validate with stakeholders

- Align expectations early to avoid scope creep

- Category	Meaning	Question to Ask
- Must	Critical	“Can we launch without this?”
- Should	Important	“Will product suffer without this?”
- Could	Nice to have	“Would it improve experience?”
- Won’t	Not now	“Should we explicitly drop this?”

# MoSCoW Example – Food Delivery App

## 🍔 Feature Prioritization

| Category | Feature | Why |
|----------|--------|-----|
| 🟥 Must Have | User login / signup | Required for user identity and ordering |
| 🟥 Must Have | Browse restaurants | Core product functionality |
| 🟥 Must Have | Place order | Without this, product cannot function |
| 🟥 Must Have | Payment system | Essential for transaction completion |
| 🟧 Should Have | Live order tracking | Improves user experience significantly |
| 🟧 Should Have | Ratings & reviews | Helps trust & decision making |
| 🟧 Should Have | Search & filters | Improves discoverability |
| 🟨 Could Have | Dark mode 🌙 | Nice UX improvement |
| 🟨 Could Have | Promo banners | Marketing visibility feature |
| 🟨 Could Have | AI food recommendations | Enhances personalization |
| ⛔ Won’t Have | AR menu preview | High effort, low priority now |
| ⛔ Won’t Have | Social food feed | Not aligned with MVP scope |
| ⛔ Won’t Have | Gamification badges | Nice idea, but not needed now |
