## RICE framework

📌 **What is RICE?**
RICE is a prioritization framework used to score and rank features based on impact.

It stands for:

R → Reach (How many users will it impact?)
I → Impact (How much value does it create?)
C → Confidence (How sure are you about impact?)
E → Effort (How much work is needed?)

**RICE Formula**
RICE Score = (Reach × Impact × Confidence) / Effort

⏰ **When to Use RICE**

**Use RICE when:**

- You have multiple feature ideas
- You need data-driven prioritization
- Stakeholders disagree on priorities
- You are planning a roadmap or sprint
- You want to avoid “gut-feel” decisions

👉 Best for: mid-stage product planning

📍 **Where to Use RICE**
📦 Product roadmaps
🧾 Backlog prioritization
🏃 Sprint planning
📊 Feature comparison docs
🤝 Stakeholder alignment meetings

⚙️ **How to Use RICE**
- Step 1: List features
- Step 2: Assign scores

  - Reach → number of users affected
  - Impact → value scale (0.25, 0.5, 1, 2, 3)
  - Confidence → % certainty (0–100%)
  - Effort → person-months or dev effort

- Step 3: Calculate score
- Step 4: Rank features by score


# RICE Prioritization – Food Delivery App

| Feature               | Reach (users/month) | Impact | Confidence | Effort (person-months) | RICE Score |
|----------------------|---------------------|--------|------------|-------------------------|------------|
| Live order tracking  | 50,000              | 2      | 80%        | 3                       | 26.67      |
| Dark mode            | 30,000              | 0.5    | 90%        | 1                       | 13.5       |
| AI recommendations   | 40,000              | 1      | 60%        | 5                       | 4.8        |
| Better search filters| 60,000              | 1      | 70%        | 2                       | 21         |

🧠 **Interpretation**
Feature	Insight

- Live tracking	High impact → top priority
- Search filters	High reach + good ROI
- Dark mode	Nice UX, but low impact
- AI recommendations	High effort + low confidence → lower priority
