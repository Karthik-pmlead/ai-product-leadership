# 🎯 5Es Framework (Customer Experience Model)

## 📌 What is it?

The **5Es framework** is a customer experience model used to understand the full user journey from first touch to long-term engagement and retention.

It helps PMs design **end-to-end product experience loops**, not just conversions.

---

## 🧠 5Es Stages

- 🎯 Entice  
- 🚪 Enter  
- 🤝 Engage  
- 🚪 Exit  
- 🔁 Extend  

---

## ⏰ When to Use 5Es

Use 5Es when:

- Designing user journeys
- Improving onboarding experience
- Optimizing retention
- Building consumer apps or SaaS products
- Analyzing end-to-end product experience

---

## 📍 Where to Use It

- Product UX design
- Onboarding flows
- Customer journey mapping
- Retention strategy
- Product experience audits

---

## ⚙️ 5Es Framework Table

| Stage   | Meaning | User Mindset | PM / Product Action |
|----------|--------|--------------|----------------------|
| 🎯 Entice | Attract users | "This looks interesting" | Ads, SEO, social media, branding |
| 🚪 Enter  | First interaction | "Let me try this" | Landing page, signup, onboarding |
| 🤝 Engage | Active usage | "This is useful" | Core features, product experience |
| 🚪 Exit   | End session | "I’m done for now" | Smooth exit, session handling |
| 🔁 Extend | Re-engagement | "I should come back" | Push notifications, email, retention loops |

---

## 🍔 Example (Food Delivery App)

| Stage | Example |
|------|--------|
| Entice | Instagram ad showing ₹99 meals |
| Enter | User downloads app and signs up |
| Engage | Browses restaurants and places order |
| Exit | Completes order and closes app |
| Extend | Push notification: “Your favorite restaurant is live” |

---

## 🧠 Simple Mental Model

- 🎯 Entice → Get attention  
- 🚪 Enter → First interaction  
- 🤝 Engage → Core usage  
- 🚪 Exit → Clean completion  
- 🔁 Extend → Bring users back  

---

## 🚀 PM Insight

> Great products don’t stop at conversion — they design for **continuous engagement loops**

5Es helps you think beyond funnels into **lifecycle experience design**.
