# Support Guide

## Overview

This document helps support teams answer common questions, troubleshoot issues, and provide consistent responses.

## Product details

| Field | Value |
|---|---|
| Product | Guided onboarding and lifecycle nudges |
| Owner | Support Lead |
| Date | 2026-06-12 |
| Audience | Support, CX, success teams |

## Common issues

| Issue | Likely cause | Suggested response |
|---|---|---|
| User cannot find the checklist | Feature not enabled or user not eligible | Confirm rollout status and eligibility. |
| User is not receiving reminders | Email preferences or send limits | Check messaging settings and frequency caps. |
| Checklist is not completing | User skipped required step | Walk them through the missing setup action. |

## Troubleshooting steps

| Step | Details |
|---|---|
| 1 | Confirm the user is in the rollout cohort. |
| 2 | Check whether onboarding events are tracked correctly. |
| 3 | Verify email preferences and delivery logs. |
| 4 | Escalate to product or engineering if needed. |

## Escalation path

| Type of issue | Escalate to |
|---|---|
| Bug or UI issue | Engineering |
| Tracking issue | Data / Analytics |
| Messaging issue | Marketing Ops |
| Access or entitlement issue | Support Ops |

## Response templates

### “I do not see the checklist.”

Please confirm you are a new self-serve user in the rollout cohort. If the feature should be available and still does not appear, we’ll investigate further.

### “I am not getting reminders.”

Please check your email preferences first. If reminders are enabled and you still do not receive them, we will review delivery logs.

## Notes

- Keep responses short and consistent.
- Use plain language.
- Update the guide when the product or rollout changes.
