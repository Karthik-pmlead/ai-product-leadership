# Discovery Templates

Reusable templates for product discovery, framing, validation, and early alignment.

## Navigation

| Template | Purpose | File |
|---|---|---|
| Product Proposal | Frame an early idea before a full PRD. | [product-proposal.md](./product-proposal.md) |
| Product Brief | Summarize the problem, audience, and approach in a lightweight format. | [product-brief.md](./product-brief.md) |
| One-Pager | Communicate the core idea in a short, easy-to-scan format. | [one-pager.md](./one-pager.md) |
| Research Summary | Synthesize user, market, or competitive research findings. | [research-summary.md](./research-summary.md) |
| Experiment Plan | Define the hypothesis, setup, metrics, and success criteria. | [experiment-plan.md](./experiment-plan.md) |
| PRD | Define the problem, goals, scope, requirements, and success metrics. | [prd.md](./prd.md) |
| PRD Pitch | Present the product idea and get early stakeholder alignment. | [prd-pitch.md](./prd-pitch.md) |

## What’s inside

| File | Best for |
|---|---|
| `product-proposal.md` | Early-stage idea framing and initial buy-in. |
| `product-brief.md` | Lightweight alignment before detailed planning. |
| `one-pager.md` | Fast executive or stakeholder review. |
| `research-summary.md` | Turning research into actionable insights. |
| `experiment-plan.md` | Testing assumptions before committing to scale. |
| `prd.md` | Full product definition and delivery alignment. |
| `prd-pitch.md` | Selling the idea before writing the full PRD. |

## Suggested use order

1. Start with `product-proposal.md` or `one-pager.md` to frame the idea.
2. Use `research-summary.md` to ground the idea in evidence.
3. Use `experiment-plan.md` to validate assumptions.
4. Use `prd-pitch.md` to get early stakeholder alignment.
5. Use `prd.md` to define the final product requirements.

## Notes

- Keep discovery artifacts short and hypothesis-driven.
- Use examples where the template needs context.
- Update documents as you learn more.
