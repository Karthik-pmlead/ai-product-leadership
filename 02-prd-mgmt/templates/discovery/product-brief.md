# Product Brief

## Overview

This document is a lightweight one-pager that explains the opportunity, evidence, success metrics, and scope boundaries.

## Opportunity

| Item | Details |
|---|---|
| Problem | New users do not reach first value quickly enough. |
| Audience | New self-serve users. |
| Why prioritize | Improving activation should improve retention and reduce support load. |

## Supporting evidence

| Evidence | Detail |
|---|---|
| User feedback | Users say setup feels confusing. |
| Behavior data | Most drop off before completing the first setup step. |
| Support data | Setup questions are the most common ticket category. |

## Proposed approach

| Item | Details |
|---|---|
| Solution | Add a checklist, better onboarding copy, and guided setup. |
| Expected value | Faster time to value and improved retention. |

## Success metrics

| Metric | Baseline | Target |
|---|---:|---:|
| Activation rate | 32% | 50% |
| Time to first value | 12 minutes | 6 minutes |
| D7 retention | 18% | 25% |

## Non-goals

| Non-goal | Reason |
|---|---|
| Enterprise onboarding | Not the priority segment. |
| Major UI redesign | Too broad for this brief. |

## Notes

- Keep it short.
- Focus on why this matters and what success looks like.
