# Launch Templates

Reusable templates for product launch communication, customer education, and release announcement.

## Navigation

| Template | Purpose | File |
|---|---|---|
| Press Release | Write the external launch story and key value proposition. | [press-release.md](./press-release.md) |
| Launch Email | Communicate launch details to users, customers, or internal stakeholders. | [launch-email.md](./launch-email.md) |
| Release Notes | Summarize shipped changes, fixes, and improvements. | [release-notes.md](./release-notes.md) |
| FAQ | Capture common launch questions and objections. | [faq.md](./faq.md) |

## What’s inside

| File | Best for |
|---|---|
| `press-release.md` | Public announcement and media-ready product story. |
| `launch-email.md` | Email communication to users, customers, or internal teams. |
| `release-notes.md` | Clear summary of what changed and why it matters. |
| `faq.md` | Preempting questions, objections, and support load. |

## Suggested use order

1. Finalize the product story in `press-release.md`.
2. Translate that story into `launch-email.md` for your audience.
3. Publish `release-notes.md` with the actual shipped changes.
4. Add `faq.md` to help support, sales, and users understand the launch.

## Notes

- Keep launch messaging consistent across all assets.
- Write for the audience of each document, not one generic reader.
- Use examples to show the intended tone and level of detail.
