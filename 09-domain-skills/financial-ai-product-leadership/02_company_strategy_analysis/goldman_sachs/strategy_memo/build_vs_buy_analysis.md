# Build vs Buy Analysis — Marquee AI Stack

## 1. Evaluation Criteria

- Data sensitivity
- Regulatory constraints
- Workflow integration depth
- Competitive differentiation
- Customization requirements

---

## 2. Core Stack Components

### 2.1 LLM Layer
Recommendation: BUY (foundation models)
- OpenAI / internal LLM infrastructure

Reason:
- Commodity capability layer
- Rapidly evolving ecosystem

---

### 2.2 Retrieval (RAG) Layer
Recommendation: BUILD

Reason:
- Must integrate proprietary Goldman research data
- Requires domain-specific ranking logic
- Critical for accuracy and trust

---

### 2.3 Reasoning Layer (Multi-Agent System)
Recommendation: BUILD

Reason:
- Must encode financial domain logic
- Requires portfolio-aware reasoning
- Core differentiation layer

---

### 2.4 Context Compression Engine
Recommendation: BUILD

Reason:
- Core IP of decision intelligence system
- Deep integration with internal systems

---

### 2.5 UI / Interaction Layer
Recommendation: HYBRID

Reason:
- Leverage existing platforms (Marquee)
- Extend with AI overlays

---

## 3. Strategic Insight

Competitive advantage will not come from model selection.

It will come from:

> Proprietary context + reasoning + workflow integration layers.
