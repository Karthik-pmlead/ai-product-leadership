# 🔗 cross-functional/dependencies.md

# Dependencies

> Identifying, tracking, and resolving cross-team dependencies that impact delivery.

---

# 📌 Types of Dependencies

| Type | Example |
|---|---|
| Technical | API readiness |
| Data | Dataset availability |
| People | Shared engineering resources |
| Process | Approval workflows |
| External | Vendor / third-party systems |

---

# 🔄 Dependency Lifecycle

Identify → Log → Assign Owner → Track → Resolve → Close

---

# 📊 Dependency Management Rules

- Every dependency must have an owner
- Every dependency must have a due date
- Blockers escalated within 24–48 hrs
- No “hidden dependencies”

---

# ⚠️ Common Issues

- Late discovery of dependencies
- No clear ownership
- Cross-team misalignment
- Untracked blockers
